# Security Policy

## Supported versions

Security fixes are applied to the latest `0.6.x` release line. Older releases may
be assessed when an affected deployment cannot upgrade immediately, but they do
not receive routine fixes.

## Reporting a vulnerability

Use GitHub's private vulnerability reporting through
[Report a vulnerability](https://github.com/fermi-ad/redis-pvxs-ioc/security/advisories/new).
Do not open a public issue for suspected vulnerabilities and do not include live
credentials, private configuration, or production data in a report.

Include the affected version/revision, deployment shape, reproduction conditions,
and potential impact. Maintainers will acknowledge the report, coordinate a fix
and disclosure timeline, and publish a security advisory when appropriate.
